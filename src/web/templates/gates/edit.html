Auto-Grant Configuration New in v1.12.0

Automatically create temporary grants when users connect to servers without existing policies. Useful for opening access to trusted users without manual grant management.

Min: 1 day, Max: 365 days. Default: 7 days
0 = disabled, Max: 1440 (24 hours). Default: 60 minutes
How Auto-Grant Works
  • Trigger: User connects to server without existing grant
  • Check: Verify no revoked grant exists (admin blocked access)
  • Create: Generate temporary grant with configured duration
  • Permissions: Full access (port forwarding, no SSH login restrictions)
  • Revoke: Admin can permanently block by setting grant end_time to past
Security Note: Auto-grant only works after MFA authentication. Users must authenticate via SAML/Azure AD before auto-grants are created.
